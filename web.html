JSON (JavaScript Object Notation) is a lightweight data-interchange format and a language-independent data format. It is easy for humans to read and write. It is easy for machines to parse and generate. It is the most common data format used for asynchronous browser/server communication, largely replacing XML, and is used by AJAX.

    JSON certified expert must be able to use JSON data format and perform the following activities such as:
  • Reading and Writing JSON on the Client
  • Reading and Writing JSON on the Server
  • Using JSON in Simple AJAX Applications
  • Using JSON in AJAX Applications with jQuery and AngularJS
  • Using JSON with MongoDB
  • Using JSON in a Type-safe Manner
  • Using JSON for Binary Data Transfer
  • Querying JSON with JSONPath and LINQ
  • JSON on Mobile Platforms

XML

XML (eXtensible Markup Language) is a markup language much like HTML and was designed to store and transport data. XML is a markup language that defines a set of rules for encoding documents in a format that is both human-readable and machine-readable.

CSV

A comma-separated values (CSV) file stores tabular data (numbers and text) in plain text. Each line of the file is a data record. Each record consists of one or more fields, separated by commas. The use of the comma as a field separator is the source of the name for this file format. The CSV file format is not standardized.

    CSV certified expert must be able to use CSV data format and perform the following activities such as:
  • Serializing Existing Data
  • Modifying Existing CSVs
  • Filtering CSVs from the Command Line
  • Creating CSV files on text editors – Notepad, Microsoft Excel, OpenOffice Calc, Google Docs

Web Development – Protocols

HTTP

The Hypertext Transfer Protocol (HTTP) is an application protocol for distributed, collaborative, and hypermedia information systems. HTTP is the foundation of data communication for the World Wide Web. Hypertext is structured text that uses logical links (hyperlinks) between nodes containing text. HTTP is the protocol to exchange or transfer hypertext.

REST

Representational state transfer (REST) or RESTful web services are one way of providing interoperability between computer systems on the Internet. REST-compliant web services allow requesting systems to access and manipulate textual representations of web resources using a uniform and predefined set of stateless operations.

DDP

Distributed Data Protocol (or DDP) is a client-server protocol for querying and updating a server-side database and for synchronizing such updates among clients. It uses the publish-subscribe messaging pattern. It was created for use by the Meteor JavaScript framework.

    DDP certified expert must be able to use DDP protocol and perform the following activities such as:

  • Managing data
  • Reading the DDP stream
  • Using client-only collections
  • Implementing a multiserver DDP
